# Ogden 850 例句与中文翻译重做设计

## 背景

`/Users/wai/Desktop/ai/ogden-english` 是一个 Ogden Basic English 850 背单词网页。当前词库主要存在于 `words_data.json`，同时 `desktop_words.html` 内嵌了一份 `WORDS` 数据用于页面展示。现有例句后半部分大量使用模板句，例如 `We can talk about the ... today.`、`I can see ...`、`This one is ...`，中文说明也多为泛化解释，不利于背诵和理解。

## 目标

重做全部 850 个单词的例句内容，并为每条英文例句提供对应中文翻译。完成后网页中每个词都应有自然、短小、贴合释义的英文例句，以及可直接理解该句意思的中文翻译。

## 范围

本次修改覆盖全部 850 个词：

- 更新 `words_data.json` 中每条记录的 `使用例句`。
- 将 `words_data.json` 中每条记录的 `例句理解` 改为英文例句的中文翻译。
- 同步更新 `desktop_words.html` 内嵌的 `WORDS` 数据，确保网页直接打开时展示最新内容。
- 如技术上可行，同步更新 `Ogden_Basic_English_850_学习表.xlsx` 中对应列，避免 JSON 与表格数据源不一致。

不改变学习计划、掌握度、复习日期、页面布局或交互逻辑。

## 内容标准

每条例句遵循以下标准：

1. 英文例句自然、具体、适合初学者朗读和记忆。
2. 句子尽量简短，通常控制在 5–10 个英文词左右；必要时可以略长，但避免复杂从句。
3. 例句必须体现该词在当前中文释义中的核心用法。
4. 中文翻译直接翻译英文例句，不再使用“这个句子练习……”之类泛化说明。
5. 避免大规模重复模板句，尤其是 `We can talk about the ... today.`、`I can see ...`、`This one is ...` 这类低信息量模式。
6. 对多义词优先覆盖当前 `中文翻译` 的主要含义；如果例句只覆盖其中一个含义，中文翻译要清晰不误导。
7. 对名词注意冠词和复数；对动词注意常见搭配；对形容词注意自然语境。

## 数据方案

保持现有字段结构，不新增字段：

- `使用例句`：英文例句。
- `例句理解`：该英文例句的中文翻译。

这样可以最大限度减少前端改动。页面目前已在列表和轮播卡片中展示 `使用例句`；后续实施时会检查是否需要把 `例句理解` 也显示出来。如果当前页面没有展示中文例句翻译，会在最小范围内补充显示，避免用户看不到新增内容。

## 实施与校验

实施时会先更新数据，再验证同步结果：

1. 读取 850 条词库记录。
2. 为每条记录重写 `使用例句` 和 `例句理解`。
3. 将更新后的词库写回 `words_data.json`。
4. 替换 `desktop_words.html` 内嵌 `WORDS` 数据为同一份更新内容。
5. 尽量同步 Excel 表格对应列。
6. 运行脚本校验：
   - 总数仍为 850。
   - 每条都有非空 `使用例句` 和 `例句理解`。
   - `desktop_words.html` 内嵌数据与 `words_data.json` 一致。
   - 模板化句式数量降到 0 或接近 0。
7. 抽样检查开头、中段、后段词条，确认例句质量前后一致。

## 风险与处理

- **850 条内容量较大**：采用批量生成加脚本校验，并抽样人工检查，避免后段再次模板化。
- **释义可能已有错误或不自然**：本次优先围绕现有中文释义写例句；如发现明显错位，会修正例句以贴近常见词义，并在结果中说明。
- **HTML 内嵌数据容易不同步**：实施中以 `words_data.json` 为唯一数据源，自动替换 HTML 内嵌 `WORDS`。
- **Excel 写入可能受依赖限制**：若环境缺少写入 Excel 的依赖，会先保证网页所需 JSON/HTML 完成，并明确说明 Excel 是否已同步。

## 完成标准

任务完成时应满足：

- 全部 850 个词都有新的英文例句。
- 全部 850 个词的 `例句理解` 都是对应中文翻译。
- 网页能展示更新后的例句；如果需要，也展示中文例句翻译。
- `words_data.json` 与 `desktop_words.html` 内嵌词库一致。
- 校验脚本通过，且无明显模板化敷衍内容。
